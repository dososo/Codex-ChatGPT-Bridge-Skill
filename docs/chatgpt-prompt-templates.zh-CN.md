# ChatGPT 提示词模板

这些模板用于右侧 ChatGPT。核心要求是：ChatGPT 做规划和审查，不直接写代码、不执行 shell、不读取敏感信息。

## 任务单模板

```text
请先不要写代码。请把下面这个需求整理成一份适合交给 Codex 执行的任务单，包含：

1. 原始问题：这次真正要解决什么。
2. 预期结果：完成后应该看到什么行为。
3. 不能改变的行为：哪些接口、数据、边界和用户体验不能动。
4. 可能涉及的文件：只列受控上下文里出现的文件，不猜测 secrets、.env、cookie、私钥、完整 Connector URL 或 .git。
5. 最小修改方案：给 Codex 的实现步骤，不写具体代码。
6. 验证命令：建议 Codex 运行的最小测试。
7. 停止并询问用户的情况：需求冲突、证据不足、需要外部账号、可能读取敏感信息或可能越权时停止。
8. 可直接粘贴给 Codex 的执行提示词。

不要执行 shell，不要直接修改源码，不要输出 patch。
```

## 审查模板

```text
请审查 Codex 已完成的执行结果。不要写代码，不要给 patch，不要要求右侧 ChatGPT 执行 shell。

请重点检查：

1. 是否符合任务单。
2. 是否有 diff 风险。
3. 是否可能造成行为回归。
4. 测试是否足够。
5. 是否触碰 secrets、.env、cookie、私钥、完整 Connector URL、.git、shell 执行或源码直接修改边界。
6. 建议 Codex 下一步做什么，但只能作为 suggested_actions，必须由用户确认。
```
